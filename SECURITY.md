# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in ios-simulator-mcp, please report it by emailing ****.

**Please do not report security vulnerabilities through public GitHub issues.**

### What to Include

Please include the following information in your report:

- Type of vulnerability
- Full paths of source file(s) related to the vulnerability
- Location of the affected source code (tag/branch/commit or direct URL)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the vulnerability, including how an attacker might exploit it

### Response Timeline

- We will acknowledge receipt of your vulnerability report within 48 hours
- We will send a more detailed response within 7 days indicating the next steps
- We will keep you informed of the progress toward a fix and announcement
- We may ask for additional information or guidance

### Disclosure Policy

- Please give us reasonable time to address the issue before any public disclosure
- We will credit you in the security advisory unless you prefer to remain anonymous

## Security Best Practices

When using ios-simulator-mcp:

1. Only use with trusted iOS Simulator instances
2. Be cautious when using commands that interact with the system (file operations, AppleScript)
3. Review and validate any automated scripts that use this tool
4. Keep your Xcode and macOS up to date
5. Use in isolated development environments when possible
